## Cross-cutting risks / sequencing

- **Snapshot/delta wire format and quantization must be locked before any entity (player or enemy) can be replicated. If the netcode codec (T09) and the ECS component-registry serializer (T47) disagree on field order/quantization, every downstream replication task breaks.** (affects: netcode (T09,T10), ecs (T47,T48), enemyai (T66,T75), combat (T56)) — Make @sl/shared-types (T02) the single source of truth for REPLICATED_REGISTRY order, MsgType/EntityType enums, and quantization params. Land T09 then T47 with a shared round-trip fuzz test before T48. Resolve the OPEN quaternion-encoding question (smallest-three vs 3x f16) and position-quantization sector-bounds question at T02, not later.
- **The audio-event-id taxonomy is shared across audio, AI Director, and rendering. AI emits AudioEventIds (T68), audio resolves them (T43 manifest), netcode carries a 2-byte audioEventId in EVENT_AUDIO (T49), and rendering's FlickerBus/BeatBus keys off the same beats (T72). A drift in the id space silently desyncs scares.** (affects: audio (T43,T68), enemyai (T68), netcode (T49), rendering (T72,T73)) — Generate the AudioId union from the manifest (T43) as the canonical registry; have netcode EVENT_AUDIO and AI Director import it rather than redefine. The 2-byte id index must be stable across builds (content-hash the manifest, not the id assignment). Gate with the offline CI test (T74) that resolves every AudioId before T68/T72 integrate.
- **Host-only authority is a load-bearing invariant: physics, all AI, noise/vision grids, and the Director must run ONLY on the host, with clients pure presentation.
